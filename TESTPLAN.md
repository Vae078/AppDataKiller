# AppData 清理器 - QA 测试计划

## 1. 功能测试

### 1.1 扫描功能
| 测试项 | 步骤 | 预期结果 | 状态 |
|--------|------|----------|------|
| 正常扫描 | 点击"开始扫描"按钮 | 显示扫描进度，完成后列出AppData中的缓存文件 | ⬜ |
| 扫描Local目录 | 检查扫描结果是否包含LocalAppData路径 | 应包含 `%LOCALAPPDATA%` 下的缓存文件 | ⬜ |
| 扫描Roaming目录 | 检查扫描结果是否包含AppData路径 | 应包含 `%APPDATA%` 下的文件 | ⬜ |
| 扫描Temp目录 | 检查是否包含Temp目录 | 应包含系统Temp目录 | ⬜ |
| 扫描取消 | 扫描过程中点击"取消" | 扫描应停止，列表部分填充 | ⬜ |
| 空结果处理 | 在干净系统上扫描 | 应显示"未发现可清理文件" | ⬜ |

### 1.2 缓存识别
| 测试项 | 测试数据 | 预期识别 | 状态 |
|--------|----------|----------|------|
| Chrome缓存 | `Chrome\User Data\Default\Cache` | 应识别为Safe | ⬜ |
| VS Code日志 | `Code\logs` | 应识别为Safe | ⬜ |
| Temp文件 | `*.tmp`, `*.temp` | 应识别为Safe | ⬜ |
| 日志文件 | `*.log` | 应识别为Safe | ⬜ |
| 配置文件夹 | 包含"Config"等关键字的文件夹 | 应标记为Caution | ⬜ |
| 系统目录 | `Microsoft`, `Windows` 等 | 不应出现在列表中 | ⬜ |

### 1.3 GUI界面
| 测试项 | 步骤 | 预期结果 | 状态 |
|--------|------|----------|------|
| 列表显示 | 扫描完成后检查列表 | 显示程序名、路径、大小、风险等级 | ⬜ |
| 树形展开 | 点击展开按钮 | 显示子文件夹和文件 | ⬜ |
| 全选功能 | 点击"全选" | Safe和Caution项目被勾选 | ⬜ |
| 取消全选 | 点击"取消全选" | 所有勾选被清除 | ⬜ |
| 危险项禁用 | 查看Danger等级项目 | 复选框应被禁用，无法选择 | ⬜ |

### 1.4 清理功能
| 测试项 | 步骤 | 预期结果 | 状态 |
|--------|------|----------|------|
| 正常清理 | 选择Safe项目，点击"清理" | 文件被移入回收站 | ⬜ |
| 确认对话框 | 点击清理 | 显示确认对话框，列出待删除项 | ⬜ |
| 取消清理 | 在确认对话框点击"否" | 返回主界面，不执行删除 | ⬜ |
| 进度显示 | 清理过程中 | 显示进度条和当前处理项 | ⬜ |
| 清理取消 | 清理过程中点击"取消" | 清理停止，已处理的项进入回收站 | ⬜ |
| 空选择清理 | 不选择任何项点击清理 | 提示"请先选择要清理的项目" | ⬜ |

## 2. 安全测试

| 测试项 | 验证点 | 状态 |
|--------|--------|------|
| 黑名单保护 | `Microsoft`, `Windows` 等系统目录不被扫描 | ⬜ |
| 危险项默认不选 | Danger等级项目默认不勾选 | ⬜ |
| 回收站恢复 | 删除的文件可在回收站恢复 | ⬜ |
| 权限处理 | 无权限访问的目录不崩溃 | ⬜ |

## 3. 性能测试

| 测试项 | 标准 | 状态 |
|--------|------|------|
| 扫描速度 | 扫描AppData应在10秒内完成 | ⬜ |
| 大文件处理 | 处理>1GB的缓存文件夹不卡顿 | ⬜ |
| 内存占用 | 扫描过程中内存<200MB | ⬜ |

## 4. 兼容性测试

| 测试项 | 环境 | 状态 |
|--------|------|------|
| Windows 10 | 在Win10运行 | ⬜ |
| Windows 11 | 在Win11运行 | ⬜ |
| .NET 8运行 | 未安装.NET 8的机器 | 需安装运行时 |

## 5. 已知限制

1. 仅支持Windows系统（依赖Windows API）
2. 需要管理员权限才能清理某些系统目录
3. 回收站恢复需要在文件资源管理器中手动操作
